# NotebookLM Audio Overview Customize Prompt

貼到 NotebookLM → Studio → Audio Overview → Customize。

---

## 標準版(書/講義通用)

```
語言:繁體中文,台灣用語。

聽眾:34 歲前端工程師,職涯轉型中,興趣是戶外、約會、旅遊。
聽的情境:跑步運動時,聽過就算,不會回放。

重點聚焦:
- 這本書/講義解決什麼問題(開場 1 分鐘講清楚)
- 3 個核心概念 + 每個配一個台灣生活情境的例子
- 1 個反直覺觀點(大家以為 X,作者說 Y)
- 可套用在職涯轉型或日常的具體行動

風格:
- 開場直接切入情境,不要「大家好歡迎收聽」
- 兩個主持人要有張力,一個會反問、挑戰,不只附和
- 用類比、舉例、生活場景,避免照念原文
- 結尾給 1 個運動回家可立刻做的 takeaway

長度:15-20 分鐘。
```

## 技術書/講義版

針對程式、設計、系統等技術內容,聚焦「為什麼」多於「怎麼做」。

```
語言:繁體中文,台灣用語。

聽眾:34 歲前端工程師,熟悉 JS/React,正在補系統設計、AI、產品思維。
聽的情境:跑步,只靠聽覺吸收,無法看 code。

重點聚焦:
- 這篇講義在解決什麼工程問題
- 3 個關鍵設計決策 + 背後的 trade-off
- 1 個反直覺觀點(初階工程師會做錯的地方)
- 套用在我現在做前端/轉型時可以怎麼應用

風格:
- 避免念 code 細節,聚焦概念、類比、決策邏輯
- 一個主持人是初階工程師會問天真問題,另一個是資深的解釋
- 用建築、餐廳、開車等生活類比解釋技術概念
- 結尾給 1 個下次寫 code 可以改的習慣

長度:18-22 分鐘。
```

## 商業/自我成長書版

```
語言:繁體中文,台灣用語。

聽眾:34 歲,正在思考職涯轉型,會被「勵志套話」煩到。
聽的情境:運動時。

要求:
- 先講這本書跟同類書的差異,不要一般化
- 3 個可驗證的主張(作者的論點是什麼,不只是口號)
- 作者論證邏輯的弱點或適用邊界
- 在台灣 2026 年的現實下怎麼套用,不要照抄美國案例

風格:
- 兩個主持人要辯論,不是齊聲讚嘆
- 直接質疑書中跟現實不符的地方
- 用台灣職場、創業、生活的例子
- 結尾給 1 個下週可以試的具體實驗

長度:15-20 分鐘。
```

## 使用流程

1. 打開 https://notebooklm.google.com/
2. New notebook → 上傳 PDF / 貼 markdown 筆記
3. (選做)先用 `prompt-digest.md` 在 Claude 跑一次,產 digest 再上傳 → 對談會更聚焦
4. Studio 區塊 → Audio Overview → Customize → 貼上面版本之一
5. Generate(等 3-5 分鐘)
6. 試聽第一段,品質 OK → 右上角 3 個點 → Download
7. 存到 `~/LifeOS/podcasts/episodes/YYYY-MM-DD-[slug].mp3`
8. 跑 `node ~/LifeOS/scripts/podcast/publish-rss.js [mp3 路徑] --title "集名" --desc "一句描述"`

## 品質檢查(聽第一分鐘 + 最後一分鐘就好)

- [ ] 開場 30 秒有勾住嗎?(沒「大家好歡迎收聽」這種開場)
- [ ] 中文發音可以接受嗎?
- [ ] 兩位主持人聲音有明顯區別嗎?
- [ ] 結尾有給具體可做的 takeaway 嗎?

≥ 3 項 ✗ → 考慮切方案 B(OpenAI TTS + 自寫腳本,見 plan 檔)
