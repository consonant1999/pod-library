# Phase B-2: 書籍「好理解重寫」Prompt

**何時用**:方案 A(NotebookLM)對談品質太散漫時。先用這個 prompt 把原書壓成 digest,再丟進 NotebookLM,對談會聚焦很多。

**也可以當單獨工具用**:只想要讀書筆記(不產 podcast),跑這個 prompt 就好。

---

## Prompt 模板

```
你是我的學習夥伴。我要把 [書名] 整理成我能吸收的形式。

背景:我是 Jason,34 歲,前端工程師轉型中,興趣是戶外、約會、旅遊。
請在「我可以怎麼用」那段套用這個情境。

素材:
---
[貼上 markitdown 產出 + Obsidian 筆記]
---

用以下結構改寫(繁體中文、口語、避免 bullet 堆砌):

## 這本書解決什麼問題
(1-2 句,講人話)

## 3 個核心概念
每個概念用這格式:
- **概念名**:一句話定義
- **一個生活例子**(不是書裡的,是台灣情境的)
- **為什麼這成立**(作者的論證邏輯,不只 what)

## 反直覺的地方
1-2 個顛覆常識的觀點。寫:大家以為 X,但作者說 Y,因為 Z。

## 我可以怎麼用
3 個在 [職涯轉型/戶外/約會] 可套用的行動。具體到「下週可以做」。

## 可以追問的問題
3 個作者沒回答但我會想知道的。

長度:1500-2000 字。避免照抄原文。
```

## 設計原則

1. **講清楚你是誰** → AI 的例子會貼你的情境
2. **固定 6 個 section** → 每次產出穩定,可預期
3. **要求「為什麼這成立」** → 逼 AI 講論證邏輯,不只 what
4. **要求「台灣情境的例子」** → 避免抄書裡的美國案例
5. **要求「下週可以做」** → 逼 AI 給具體行動,不是雞湯

## 變體:講義/論文版

把「這本書解決什麼問題」改成「這篇論文/講義的主張是什麼」;把「3 個核心概念」改成「論證的 3 步驟」;把「反直覺」改成「跟主流觀點的差異」。
