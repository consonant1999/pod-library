# Phase B-3: 雙人對談腳本 Prompt

**何時用**:方案 B(OpenAI TTS)主要 prompt。吃 digest 產出可直接丟 TTS 的對談腳本。

方案 A(NotebookLM)**不需要**這個 prompt — NotebookLM 會自己產對談,你只需要 `notebooklm-prompt.md`。

---

## Prompt 模板

```
把下面的 digest 改寫成 podcast 對談腳本。風格參考 NotebookLM,但更口語、有衝突感。

角色:
[Host] 好奇學習者。會提問、舉生活例、挑戰 Guest。講繁體中文。
[Guest] 熟悉這本書的專家。會解釋、舉類比、偶爾吐槽作者。講繁體中文。

規則:
1. 開頭 30 秒用具體情境/問題勾人,不要自我介紹、不要「歡迎收聽」
2. Host 要反問,不是只會「對對對」
3. Guest 要舉自己的例子或類比,不是念原文
4. 自然口語:「對」「欸」「所以意思是」「我懂了但是...」
5. 每段 ≤ 80 字(TTS 好切、自然停頓)
6. 中間 1-2 次有意義的辯論(不是一致同意)
7. 結尾給 1 個 actionable takeaway
8. 目標:3000-3500 繁體中文字

格式(嚴格遵守,TTS 要解析):
[Host]: ...
[Guest]: ...

素材:
---
[貼上 digest]
---

開始。
```

## 設計原則

1. **人設要有衝突**(Host vs Guest)→ 避免齊聲讚嘆的無聊對話
2. **明禁套話**(「歡迎收聽」)→ 省開場 30 秒
3. **每段 ≤ 80 字**→ TTS 才有自然停頓,也方便逐段重生
4. **中間要辯論**→ 提升聽覺注意力,運動中才不會恍神
5. **結尾要 takeaway** → 吸收成果,不是聽完就忘

## 品質 check(產完腳本後掃一遍)

- [ ] 開頭有沒有「大家好/歡迎收聽」?有 → 刪掉
- [ ] Host 的台詞有沒有超過 3 次「對」「對啊」「沒錯」?有 → 加反問
- [ ] Guest 有沒有舉至少 2 個類比/自己的例子?沒有 → 退回重產
- [ ] 最後一段是不是具體行動?如果是抽象總結 → 退回重產
- [ ] 總字數在 3000-3500?太少 → 要求加具體例子
